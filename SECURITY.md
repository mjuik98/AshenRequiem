# Security Policy

## Reporting

If you discover a security issue in this repository:

- Prefer GitHub's private vulnerability reporting flow when it is enabled for the repository.
- If private reporting is not available, open a minimal GitHub issue without exploit details and request a private follow-up channel.

Please avoid posting working exploit steps, secrets, or proof-of-compromise data in a public issue.

## Scope

Reports are most useful when they include:

- the affected commit or release
- reproduction steps
- expected impact
- any mitigation or temporary workaround you identified
