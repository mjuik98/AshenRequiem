# Ashen Requiem (Vamplike)

> HTML / JavaScript 기반의 뱀서라이크(Vampire Survivors-like) 프로젝트입니다.
> MVP 검증을 마치고 **본격적인 기능 확장 및 폴리싱(Phase 2)** 단계로 진입했습니다.

## 프로젝트 비전 및 목표

브라우저 환경에서 동작하는 성능 좋고 확장성 있는 뱀서라이크 게임을 만듭니다.
단순한 형태의 **이동 → 자동 공격 → 레벨업** 루프(MVP)를 완성하였으며, 앞으로는 **보스 몬스터, 메타 진행(영구 성장), 세이브/로드 기능, 다채로운 무기 시너지**를 추가하여 게임의 깊이를 더해갑니다.

---

## 엔진 아키텍처 개요

본 프로젝트는 무거운 범용 엔진 대신 아래와 같은 명확한 **책임 분리 원칙**을 기반으로 자체 구축되었습니다.

- **Scene**: 게임의 흐름 제어 (메인 화면 ↔ 플레이 ↔ 결과)
- **System**: 게임 규칙 로직 전담 (이동, 스폰, 충돌, 데미지 등)
- **Entity**: 얇은 데이터/상태 객체
- **Renderer / UI**: 규칙 개입 없이 화면 출력 및 입력 전달 전담

이 아키텍처를 바탕으로 데이터 베이스(`src/data/`) 확장만으로도 새로운 몬스터와 무기, 업그레이드를 손쉽게 추가할 수 있습니다.

> 🛠 **AI 에이전트 및 개발자 가이드**: 
> 상세한 시스템 파이프라인, 코딩 컨벤션, 폴더 구조 등은 [AGENTS.md](./AGENTS.md) 시크릿 가이드를 엄격하게 따릅니다.

## 🚀 로드맵 (Roadmap)

### Phase 1: MVP (완료 ✅)
- [x] 플레이어 및 적 추적 이동
- [x] 자동 공격 로직 및 투사체 발사
- [x] 적 처치 및 경험치 획득 루프
- [x] 레벨업 시 3개 선택지 제공
- [x] 시간 경과에 따른 적 스폰량 증가 (웨이브 시스템)

### Phase 2: 본격 확장 (현재 진행 중 🚧)
- **메타 진행 (Meta-Progression)**: 런(Run) 종료 후 얻은 자원으로 영구적인 능력치 상승 기능 추가
- **세이브 & 로드 (Save / Load)**: `sessionState` 및 `localStorage`를 활용한 기록 및 진행도 저장
- **보스전 (Boss Encounters)**: 특정 시간에 등장하는 강력한 체력과 고유 패턴/페이즈를 가진 엘리트 및 보스 설계
- **무기 시너지 강화**: 다중 무기 조합에 의한 융합(Synergy) 시스템 고도화
- **정교한 상태 이상**: 화상, 빙결, 기절 등의 도트 틱(Tick) 및 군중 제어(CC) 최적화
- **연출 및 사운드 폴리싱**: 피격 이펙트, 타격음 개선 및 파티클 시스템 최적화

## 개발 환경 및 실행 방법

### 로컬 서버 실행
프로젝트는 Vite를 번들러 및 개발 서버로 사용합니다.

```bash
npm install
npm run dev
```
(브라우저에서 `http://localhost:5173` 으로 접속)

### 테스트 및 검증
주요 시스템들의 무결성과 데이터 정합성을 검증하기 위한 자동화된 테스트가 준비되어 있습니다.

```bash
# 데이터 무결성 검증 (순환 참조, 잘못된 ID 등)
npm run validate

# 전체 단위 테스트 실행 (Node.js Test Runner)
npm test

# 빌드 후 실제 브라우저에서 deterministic smoke 실행
npm run test:smoke

# 단위 테스트 + smoke까지 한 번에 검증
npm run verify
```
